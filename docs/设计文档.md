## 1. 用户分析

### 1.1 用户角色定义

| 角色 | 描述 | 核心场景 |
|------|------|----------|
| **访客（Visitor）** | 未登录用户，仅浏览首页 | 访问首页、查看产品介绍、注册账号 |
| **普通用户（End User）** | 已登录用户，使用智能客服问答 | 提问、查看历史、反馈、管理会话 |
| **知识库管理员（Knowledge Admin）** | 管理知识库文档和 RAG 配置 | 上传文档、查看切片、调整参数、版本管理 |
| **系统管理员（System Admin）** | 管理系统用户和配置 | 用户管理、系统配置、查看统计、审计日志 |

### 1.2 用户痛点与需求

**普通用户痛点**：
1. 传统客服响应慢（人工客服排队）
2. 答案不准确（人工记忆有限）
3. 无答案溯源（不知道答案从哪来）
4. 无法 7x24 小时服务

**知识库管理员痛点**：
1. 文档更新繁琐（需重新上传）
2. 不知道用户常问什么（缺少统计）
3. 切片效果难调优（参数不透明）

**系统管理员痛点**：
1. 无法了解系统使用情况（缺少报表）
2. 用户权限管理复杂（缺少 RBAC）

### 1.3 用户旅程地图

**普通用户旅程**：
```
访问首页 → 注册/登录 → 进入聊天界面 → 输入问题 → 查看答案 → 查看来源
→ 反馈（点赞/点踩）→ 继续提问 → 查看历史记录 → 退出登录
```

**知识库管理员旅程**：
```
登录 → 进入知识库管理 → 上传文档 → 查看切片结果 → 测试问答效果
→ 调整 RAG 参数 → 查看统计报表 → 发布更新
```

### 3.3 RAG 问答数据流转

```
用户提问
  ↓ Frontend Store
Gateway (8080)
  ↓ Route to /api/chat/*
Chat Service (8081)
  ↓ 1. Embedding API（BAAI/bge-m3）
向量嵌入
  ↓ 2. Redis Vector Search (HNSW)
返回 Top-K 切片
  ↓ 3. Prompt 构建
调用 Chat LLM (Qwen2.5-7B)
  ↓ 4. SSE 流式输出
Frontend 逐字渲染
```

### 4.3 核心模块详细设计

#### 4.3.1 RAGService.java（核心问答服务）

```java
@Service
public class RAGService {
    
    @Autowired
    private EmbeddingService embeddingService;
    
    @Autowired
    private VectorStoreService vectorStore;
    
    @Autowired
    private ChatClient chatClient;
    
    /**
     * RAG 问答主流程
     */
    public Flux<String> askStream(String question, Long kbId, Long conversationId) {
        // 1. 问题向量化
        float[] queryVector = embeddingService.embed(question);
        
        // 2. 向量检索 Top-K 切片
        List<DocumentChunk> chunks = vectorStore.search(queryVector, kbId, topK=5);
        
        // 3. 构建 Prompt
        String prompt = buildPrompt(question, chunks);
        
        // 4. 调用 LLM 流式生成
        return chatClient.streamGenerate(prompt)
            .doOnNext(chunk -> log.info("Generated chunk: {}", chunk));
    }
    
    private String buildPrompt(String question, List<DocumentChunk> chunks) {
        StringBuilder context = new StringBuilder();
        for (DocumentChunk chunk : chunks) {
            context.append(chunk.getText()).append("\n\n");
        }
        
        return String.format("""
            你是一个智能客服助手。请根据以下知识库内容回答用户问题。
            
            知识库内容：
            %s
            
            用户问题：%s
            
            请用中文回答，并在回答末尾标注引用的知识库片段编号。
            """, context.toString(), question);
    }
}
```

#### 4.3.2 TextChunker.java（文档切片服务）

```java
@Service
public class TextChunker {
    
    private static final int CHUNK_SIZE = 500;  // 字符
    private static final int OVERLAP = 50;      // 重叠字符
    
    /**
     * 递归字符分割
     * 优先级：段落 → 句子 → 逗号 → 强制切分
     */
    public List<String> chunk(String text) {
        List<String> chunks = new ArrayList<>();
        
        // 1. 按段落分割
        String[] paragraphs = text.split("\n\n");
        
        for (String para : paragraphs) {
            if (para.length() <= CHUNK_SIZE) {
                chunks.add(para.trim());
            } else {
                // 2. 段落过长，按句子分割
                chunks.addAll(chunkBySentence(para));
            }
        }
        
        // 3. 添加重叠
        return addOverlap(chunks, OVERLAP);
    }
```

### 4.4 技术选型依据

| 技术 | 选型 | 理由 |
|------|------|------|
| **后端框架** | Spring Boot 3.5.2 + Spring Cloud 2025.0.0 | LTS 版本，兼容 Java 17，社区活跃 |

| **前端框架** | Vue 3 + Vite + Pinia + Element Plus | 响应式 UI，构建快，组件库完善 |
| **容器编排** | Docker Compose | 一键部署，环境隔离，适合中小规模 |

## 7. 安全设计

### 7.1 JWT 认证鉴权流程

```
用户登录
  ↓ POST /api/auth/login {username, password}
User Service
  ↓ 验证密码（BCrypt 匹配）
  ↓ 生成 JWT Token
  ↓ 返回 Access Token（2h）+ Refresh Token（7d）
前端存储
  ↓ Access Token → localStorage
  ↓ Refresh Token → httpOnly Cookie
后续请求
  ↓ Header: Authorization: Bearer <Access Token>
Gateway
  ↓ 验证 Token 签名 + 过期时间
  ↓ 解析用户 ID + 角色
  ↓ 转发到下游服务（添加 X-User-Id / X-User-Role Header）
下游服务
  ↓ 从 Header 获取用户 ID + 角色
  ↓ 鉴权（@PreAuthorize 注解）
  ↓ 返回数据
Access Token 过期
  ↓ 前端用 Refresh Token 换新的 Access Token
  ↓ POST /api/auth/refresh {refreshToken}
  ↓ 返回新的 Access Token（2h）
```

**JWT Token 结构**：
```json
// Header
{
  "alg": "HS256",
  "typ": "JWT"
}

// Payload
{
  "sub": "1",                // 用户 ID
  "username": "testuser",
  "role": "USER",
  "iat": 1716451200,        // 签发时间（Unix 时间戳）
  "exp": 1716458400         // 过期时间（2h 后）
}

// Signature
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret_key
)
```

**Refresh Token 策略**：
- Refresh Token 存储在 httpOnly Cookie（防 XSS）
- Access Token 存储在 localStorage（方便前端读取）
- Refresh Token 有效期 7 天，单次使用后立即失效（旋转策略）
- 用户主动登出时，Refresh Token 加入黑名单（Redis SET）

## 8. 性能优化

### 8.1 Redis Pipeline 批量优化

**问题**：
- 向量化 1 个文档需要存储 N 个切片
- 每个 切片 需 5 次 Redis 写入（content + vector + metadata + doc_index + doc_chunks）
- **N 个切片 × 5 次写入 = 5N 次网络往返**

**解决方案**：
```java
@Service
public class VectorStoreService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    /**
     * 批量写入切片（使用 Pipeline）
     * 性能提升：55x（实测从 12s 降至 0.22s）
     */
    public void batchInsert(List<DocumentChunk> chunks, Long docId) {
        redisTemplate.executePipelined((RedisCallback<Object>) connection -> {
            for (DocumentChunk chunk : chunks) {
                String contentKey = "rag:content:doc_" + docId + "_" + chunk.getIndex();
                String vectorKey = "rag:vectors:doc_" + docId + "_" + chunk.getIndex();
                String metaKey = "rag:meta:doc_" + docId + "_" + chunk.getIndex();

                // 写入内容
                connection.stringCommands().set(
                    contentKey.getBytes(),
                    chunk.getText().getBytes()
                );

                // 写入向量（HNSW）
                connection.hyperLogLogCommands().pfAdd(vectorKey, chunk.getVector());

                // 写入元数据
                Map<String, String> metadata = Map.of(
                    "docId", docId.toString(),
                    "chunkIndex", String.valueOf(chunk.getIndex()),
                    "length", String.valueOf(chunk.getText().length())
                );
                connection.hashCommands().hSet(
                    metaKey.getBytes(),
                    metadata.entrySet().stream()
                        .collect(Collectors.toMap(e -> e.getKey().getBytes(), e -> e.getValue().getBytes()))
                );
            }
            return null;  // Pipeline 必须返回 null
        });
    }
}
```

**性能对比**：
| 操作 | 单次写入 | Pipeline | 提升 |
|------|----------|----------|------|
| 100 个切片 | 12s | 0.22s | **55x** |
| 500 个切片 | 60s | 1.1s | **55x** |

## 附录 B：关键决策记录

| 决策 | 理由 | 时间 |
|------|------|------|
| **CORS 仅由 Gateway 处理** | 双重 CORS 配置导致响应头重复，浏览器报错 | 2026-05-12 |
| **Redis key 用下划线（`rag:content:doc_36_0`）** | 曾改冒号致 NPE；Knowledge 写 / Chat 读必须一致 | 2026-05-16 |
| **stats() 从 MySQL SUM 聚合** | Redis 重启导致与 MySQL 不一致 | 2026-05-17 |
| **MinIO endpoint 用 127.0.0.1:9000** | Windows localhost 解析 IPv6 失败 | 2026-05-18 |
| **Gateway RouteConfig @Value 注入** | 支持本地和 Docker 两套配置 | 2026-05-18 |
| **前端 Nginx `proxy_buffering off`** | 否则 SSE 流中断 | 2026-05-18 |
| **DocumentService 数据库分页** | 原全表加载再 subList，大数据量 OOM | 2026-05-17 |
| **纯前端导出 + 跳过 WebSocket** | 减少后端改动范围 | 2026-05-17 |
| **Java DSL 路由替代 YAML** | YAML 静态路由在响应式 Gateway 加载失败 | 2026-05-18 |
| **Windows 文件写入用 write_file.py** | PowerShell heredoc 对 Vue/JS 不可靠 | 2026-05-23 |
